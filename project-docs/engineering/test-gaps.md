# Test Gaps To Watch

The V5 workstream/resource services and daemon integration have deterministic
coverage. Remaining risk is concentrated at live provider, restart, and
external-system boundaries.

## Highest Priority

- Repeated live daemon/provider acceptance across learning, websites,
  research, automation, and resource-based continuation over multiple days.
- Interruption at every resource-mutation and finalization journal boundary,
  including manual recovery inspection.
- Real external computer-use outcomes for browser, desktop, remote APIs, and
  other systems where a filesystem observation is insufficient.
- Catalog reconstruction after partial disk damage and operator validation of
  unreconstructible preferences.

## Cross-Layer Outcomes

- Completed, failed, blocked, needs-user-input, run-limit, context-limit, and
  tool-failure finalizations for both chat and system events.
- Duplicate/reordered client ingress and daemon restart during preparation,
  binding, step persistence, resource verification, or finalization.
- Attachment admission while unbound, during clarification, and when routing
  fails.
- Symlink/path escape and cross-resource mutation tests for every new
  filesystem-capable tool.
- Context-pressure compaction without losing request/resource ownership.

When a change crosses packages, run both package suites and inspect the SQLite
rows, context repository, real resources, feedback summary, and Git history.
