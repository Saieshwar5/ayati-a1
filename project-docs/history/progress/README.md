# Progress

Use this directory for project history and implementation records.

Suggested structure:

- `current-state.md`: what works now, what is incomplete, and what should happen next.
- `changelog.md`: short chronological summaries.
- `commits/`: detailed commit records using project IDs such as `AYATI-0001`.
- `milestones/`: larger phase summaries.

Progress files should explain what changed, why it changed, decisions made, testing done, and follow-ups.
